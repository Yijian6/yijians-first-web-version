# Minecraft 思考空间 · 使用说明

这个文件夹就是你的写作空间。你在这里写 Markdown，网站上的世界会自动长出来。

## 一、Obsidian 初次设置（只做一次）

1. 打开 Obsidian → 左下角「打开另一个仓库」→「打开本地文件夹作为仓库」
2. 选择这个文件夹：`C:\Users\觉\Desktop\self_web\content\minecraft`
3. 完成。以后写作直接打开这个仓库。

## 二、文件的组织方式

```
content/minecraft/
  数据结构/                        ← 一个文件夹 = 一个领域 = 世界里的一栋楼
    _领域.md                       ← 这个领域的说明和蓝图
    2026-07-21-链表与数组之争.md    ← 一篇文章 = 一层楼
    assets/                        ← 这个领域的配图都放这里
```

**新开一个领域**：新建文件夹，在里面建一个 `_领域.md`，照抄下面的格式：

```
---
名称: 数学
英文: Math
排序: 2
---

## 蓝图

- [ ] 极限的直觉
- [ ] 导数是什么
```

- `英文` 用于生成网址（比如 Math → `/mc/math/`），必填
- `排序` 决定这栋楼在天际线里的位置，数字小的靠前
- 蓝图是可选的：写上你计划写的主题，没写的会显示成半透明的「规划中」楼层——这是给自己的公开承诺

## 三、写一篇文章

1. 在领域文件夹里新建笔记，文件名建议 `2026-07-21-文章标题` 的格式
2. 在 Obsidian 右上角属性面板（或输入 `---`）添加属性：

| 属性 | 必填？ | 说明 |
|------|-------|------|
| `日期` | ✅ | 决定这层楼的位置（日期类型，选一下就行） |
| `简介` | 可选 | 一句话介绍，会显示在列表里 |
| `蓝图` | 可选 | 认领蓝图里的一项（文字要和蓝图完全一致），认领后那层「规划中」就变成实心楼层 |
| `草稿` | 可选 | 设为 `true` 时不发布；写完删掉这个属性 |

3. 正文第一行写 `# 文章标题`（这就是网页上的大标题）
4. 配图放进 `assets/` 文件夹，直接粘贴 `![[图片名.png]]` 即可（文件名带空格也没问题）

### 支持的 Obsidian 写法

| 写法 | 效果 |
|------|------|
| `![[图.png]]` / `![[图.png\|300]]` | 图片 / 按 300px 宽度显示 |
| `[[另一篇文章]]` | 站内链接（按文章标题或文件名匹配，先找同领域再找其他领域） |
| `[[另一篇文章\|点这里]]` | 站内链接，显示"点这里" |
| `==高亮文字==` | 金绿色高亮 |
| 单次换行 | 和 Obsidian 一样，就是换行 |
| 行首 Tab 缩进 | 显示为带左边线的缩进段落（不会变成代码块） |
| 标题、**加粗**、引用、列表、表格、``` 代码块 | 都会变成网站的样式 |

注意：`[[内链]]` 指向的文章如果不存在或还是草稿，发布时会报错提醒你，防止上线死链。

## 四、发布（每次写完）

1. 确认文章没有 `草稿: true` 属性
2. 回到网站文件夹 `C:\Users\觉\Desktop\self_web`，**双击「发布文章.cmd」**
3. 等它自动跑完（约 2-5 分钟，包含全套质量检查），看到绿色的「✅ 已上线」就成功了
4. 打开 https://yijian6.pages.dev/minecraft.html 看你的楼长高了一层

## 五、常见问题

**报错「日期缺失或格式不对」** → 去 Obsidian 属性面板检查 `日期` 属性，格式是 2026-07-21。

**报错「找不到图片」** → 检查图片是否放在 `assets/` 里，文件名（包括后缀）和正文里写的完全一致。

**报错「图片超过 1.2MB」** → 用 https://squoosh.app 压缩一下（导出 WebP 格式），替换原图。

**报错「以下文件有未提交的改动」** → 说明网站代码有别的改动混进来了。打开 Claude Code 说一句「帮我处理未提交的改动」。

**发布中途失败** → 把报错信息复制给 Claude Code，它能帮你定位。内容不会丢，修好后重新双击发布即可。

**想删掉示例文章** → 直接删除「数据结构」文件夹里的三篇示例 md（或整个文件夹），下次发布时网站会同步。

## 六、和 Obsidian 插件共存

- 你可以把自己习惯的 `.obsidian` 配置复制进来，它不会被发布
- **下划线开头的文件夹会被构建忽略**：模板、日记这类插件的目标文件夹请命名为 `_模板`、`_日记`，它们不会被当成领域
- 仓库根目录的散装笔记（不在领域文件夹里的）也不会被发布

## 七、几条约定

- 别改动 `content/minecraft/` 之外的网站文件（那些交给 Claude Code）
- `_领域.md` 的蓝图列表尽量别删已认领的项（会导致构建报错，报错信息会告诉你原因）
- 文章质量优先：这个空间的意义是沉淀思考，不是堆数量
